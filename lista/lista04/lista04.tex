\item Escreva uma versão recursiva do algoritmo \texttt{\blue particiona}.
    
\item     5. Analise e discuta a versão abaixo do algoritmo Particiona, ela funciona? Para um mesmo vetor A[] essa versão e a versão anterior tem o mesmo resultado, ou seja, na saída de ambos os algoritmos o vetor A[] fica idêntico e é encontrada a mesma posição para pivô ?

\begin{verbatim}
algoritmo particiona2( A, p, r )
   x <- A[r]
   i <- p
   para j de p até r - 1 faça
      se A[j] <= x então 
         troca(A[i], A[j])
         i <- i + 1
      fim-se
   fim-para
   troca(A[i], A[r])
   retorne i
fim.    
\end{verbatim}

\item Escreva o pseudocódigo (ou implementação) de uma versão iterativa do algoritmo de ordenação \texttt{QuickSort}. (\textbf{Dica:} Utilize a estrutura de dados \textit{pilha}) 
\pagebreak

\item Analise e discuta a versão abaixo do algoritmo QuickSort, ela funciona? O que acontece se trocar a linha ``enquanto $p < r$ faça'' por ``se $p < r$ então'' ?
\begin{verbatim}
algoritmo QuickSort( A, p, r )
   enquanto p < r faça
      j <- Particiona(A, p, r)
      QuickSort(A, p, j-1)
      p <- j + 1
   fim-enquanto
fim.
\end{verbatim}
